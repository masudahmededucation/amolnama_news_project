================================================================================
FEATURES
================================================================================

1. PAGE SIZES (PAGE_SIZES map — 10 options)
   Standard:
     - 5×7″ Portrait, 7×5″ Landscape
     - 4×6″ Portrait, 6×4″ Landscape
     - 8×10″ Portrait, 10×8″ Landscape
   Paper:
     - A4 Portrait (8.27×11.69″), A4 Landscape
     - Letter Portrait (8.5×11″), Letter Landscape
   All rendered at 300 DPI for print quality

2. LAYOUTS (14 layout options)
   - 2h:   2 Side-by-Side (default)
   - 3h:   3 Side-by-Side
   - 1:    1 Full Page
   - 4g:   2×2 Grid (4 photos)
   Multi-row grids (generated by makeGrid(cols, rows) helper):
   - 2x3:  2×3 Grid (6 photos)  — heightRatio 1.5
   - 2x4:  2×4 Grid (8 photos)  — heightRatio 2
   - 2x5:  2×5 Grid (10 photos) — heightRatio 2.5
   - 2x6:  2×6 Grid (12 photos) — heightRatio 3
   - 2x7:  2×7 Grid (14 photos) — heightRatio 3.5
   - 2x8:  2×8 Grid (16 photos) — heightRatio 4
   Asymmetric:
   - 1l2r: 1 Large Left + 2 Right
   - 2l1r: 2 Left + 1 Large Right
   - 1t2b: 1 Top + 2 Bottom
   - 2t1b: 2 Top + 1 Bottom
   Layout buttons show CSS-grid thumbnails previewing the arrangement
   Tall layout buttons use .album-layout-btn--tall class
   heightRatio multiplies page height for multi-row grids

3. FRAME SLOTS
   - Dotted-line boxes matching layout positions
   - Click to browse files or drag & drop images onto any frame
   - Each frame has hidden <input type="file" accept="image/*">
   - Images displayed with cover-fit positioning (manual absolute position)
   - Frames preserve images when switching layouts (up to frame count)

4. IMAGE REPOSITIONING (drag-to-pan)
   - After dropping an image, user can drag it inside the frame to reposition
   - Mouse drag + single-finger touch supported
   - framePans[idx] stores { x, y } offset in display pixels from center
   - Clamped so image always covers the frame (no empty gaps)
   - .panning class shows grabbing cursor during drag
   - touch-action: none prevents browser scroll/zoom while panning
   - Pan offset is preserved through layout/size changes
   - Pan offset is correctly translated to source-image coordinates for export
   - rAF-throttled panning for smooth 60fps performance

4b. PER-FRAME ZOOM
   - Zoom in (+) / zoom out (−) buttons on each filled frame toolbar
   - Mouse wheel on frame also zooms (deltaY direction)
   - frameZooms[idx] stores zoom multiplier (1.0 = no zoom, range 1–3)
   - Zoom badge shows current level (e.g. "1.2×") when > 1
   - Zoom multiplies cover-fit dimensions — pan clamping adjusts accordingly
   - Zoom preserved through layout/size changes
   - Zoom correctly applied in full-DPI export rendering

5. IMAGE SWAP
   - Hover toolbar on each filled frame: swap (↔), zoom (+/−), and remove (✕) buttons
   - Click swap → frame highlights as "source" (orange border)
   - Click another frame → images + pans swap positions instantly
   - Escape key cancels swap mode
   - Swap hint text appears below workspace during swap

6. BACKGROUND COLORS
   - 5 preset swatches: White, Cream, Dark, Navy, Beige
   - Custom color picker (HTML5 <input type="color">)

7. BORDER & ROUNDING CONTROLS
   - Frame Padding slider: 0–40px (default 12) — gap between frames
   - Corner Rounding slider: 0–40px (default 0) — border-radius on frames
   - Rounding updates instantly without full rebuild

8. CREATE ALBUM (two-step flow)
   - "Create Album" button → renders full-DPI canvas in background
   - Uses setTimeout(50) to let UI update before heavy rendering
   - Shows result section with canvas preview + download buttons
   - Any settings change hides the result (forces re-create)

9. DOWNLOAD (3 formats)
   - JPG: canvas.toBlob('image/jpeg', 0.95)
   - PNG: canvas.toBlob('image/png')
   - PDF: jsPDF with correct page orientation and dimensions
   - All blob URLs revoked after 3s

10. RESET
    - Revokes all blob URLs, clears images + pans + zooms, hides result, rebuilds

11. RESPONSIVE
    - Canvas workspace auto-scales to container width (max 800px)
    - Window resize triggers debounced (200ms) relayout
    - Mobile: controls stack vertically, buttons go full-width

12. USE CASES (mentioned in subtitle)
    - Photo album pages
    - NID card front/back print layout
    - Passport photo print sheets
    - Family/travel photo collections (multi-row grids to share many photos in one file)

13. PERFORMANCE
    - rAF-throttled panning for smooth 60fps drag
    - createImageBitmap for non-blocking image decode
    - Frame-by-frame canvas rendering (one frame per rAF tick) to prevent jank
    - setTimeout yield before heavy rendering operations

================================================================================
KEY FUNCTIONS
================================================================================

init()                         — boots layout buttons, bindings, initial render
buildLayoutButtons()           — attaches click handlers to layout selector buttons
applyLayout()                  — calculates display scale, rebuilds all frame DOM
createFrameElement(idx, dW, dH) — builds one frame with positioned image + toolbar + zoom
getCoverDimensions(img, fw, fh, zoom) — calculates cover-fit width/height with zoom multiplier
clampPan(offset, imgDim, frameDim) — clamps pan so image covers frame
bindFramePan(frame, idx)       — attaches mouse + touch drag for repositioning
startPanDrag / onPanDragMove / endPanDrag — pan drag lifecycle
applyFrameStyles()             — updates border-radius without full rebuild
loadImageToFrame(idx, file)    — creates blob URL, loads Image, rebuilds frame
rebuildFrame(idx)              — replaces single frame DOM, re-sorts z-order
removeImage(idx)               — revokes URL, nulls image + pan, rebuilds frame
applyFrameZoom(idx, delta)     — applies zoom delta to frame, clamps 1–3, rebuilds
startSwap / completeSwap / cancelSwap — image swap between frames
createAlbum()                  — triggers rAF frame-by-frame renderCanvas + showResult
renderCanvas()                 — renders full-DPI canvas with all frames + pans + zoom
drawCoverWithPan(ctx, img, ..., zoom) — cover-fit draw with pan + zoom translation
makeGrid(cols, rows)           — generates 2-column grid frame coordinates
roundedRect(ctx, ...)          — draws rounded rectangle path for clipping
showResult() / hideResult()    — toggles result section visibility
downloadAs(format)             — routes to JPG/PNG/PDF download
downloadAsPdf(baseName)        — jsPDF generation with correct orientation
triggerDownload(blob, filename) — blob URL download + revoke
resetAll()                     — full cleanup and relayout

================================================================================
STATE VARIABLES
================================================================================

currentPageSize    — key into PAGE_SIZES (default '5x7')
currentLayout      — reference to LAYOUTS entry (default LAYOUTS[0] = 2h)
bgColor            — background fill color (default '#FFFFFF')
borderPx           — frame padding in display pixels (0–40, default 12)
roundPx            — corner rounding in display pixels (0–40, default 0)
frameImages        — array of Image objects (or null) parallel to layout frames
frameObjectUrls    — array of blob URLs to revoke on cleanup
framePans          — array of { x, y } pan offsets in display pixels
frameZooms         — array of zoom multipliers (1.0 = no zoom, range 1–3)
swapSourceIdx      — index of frame being swapped from (-1 = none)
displayScale       — ratio of display size to print pixel size
panDragIdx         — index of frame currently being panned (-1 = none)
panDragStartX/Y    — mouse/touch start position for current pan drag
panDragOrigX/Y     — pan offset at start of current drag
resultCanvas       — rendered full-DPI canvas (null until Create Album)
