================================================================================
BOOKWRITER — DB MAPPING: REFERENCE LIST DISPATCHER
================================================================================
Module:       views_api_refs.py (standalone, single-file module)
Tables:       Every ref_* table (14 total) — read-only
Templates:    Not directly consumed yet — UI uses inline server-rendered
              dropdowns. Available for any future client-side drop-in.
Common steps: See db-mapping-common-steps.txt

================================================================================
ENDPOINT
================================================================================
GET /api/refs/<ref_group_code>/

View:        api_bookwriter_ref_list
Auth:        NONE (public — anonymous-OK)
Throttle:    None applied (read-only, small payload, idempotent)

URL parameter:
  ref_group_code  one of REF_GROUP_TABLE_MAP keys (see below)

Response (success): {
  ok:    true,
  group: '<ref_group_code>',
  items: [
    {id: <pk_int>, code: '<code>', name_en: '<en label>', name_bn: '<bn label>'},
    ...
  ]
}
- name_bn key is OMITTED for tables that have no _name_bn column
  (cover_palette / cover_background / cover_font / view_referrer / view_device)
- Items ordered by sort_order then PK ascending
- Filtered to is_active=True
- chapter_status group additionally excludes 'outline' code (until v02 SQL runs)

Response (unknown group):
HTTP 404 + body { ok: false, error: 'Unknown ref group',
                  available_groups: [<sorted list of every key>] }

================================================================================
DESCRIPTIVE / METADATA COLUMNS (read-only, NOT in the dispatcher response)
================================================================================
Several ref tables carry extra columns the dispatcher does NOT include in
its JSON response. They are loaded only when a server-side template needs
them (e.g. for tooltips, scheduler logic, or chip styling). Documented
here so they don't appear "unwired" in coverage audits.

  RefBookStatus.book_status_description           Long-form description for
                                                    a future "what each status
                                                    means" tooltip on the
                                                    book metadata picker.
  RefChapterVisibility.chapter_visibility_description
                                                  Same pattern — explains
                                                  private/beta/public scope.
  RefBetaPermission.beta_permission_description    Explains read/comment/suggest
                                                    in the share modal tooltip.
  RefSerialReleaseStatus.serial_release_status_chip_color_hex
                                                   Hex color for the publish
                                                   dashboard's status chip
                                                   (rendered server-side per
                                                   row when the dashboard ships).
  RefPublishCadence.publish_cadence_interval_days  Scheduler config — how often
                                                   to drop the next chapter.
  RefPublishCadence.publish_cadence_weekday        Day-of-week (0=Mon..6=Sun)
                                                   for weekly cadences.
  RefCoverFont.cover_font_family_css               Already documented in
                                                   db-mapping-cover-design.txt
                                                   under REF TABLES (4)
  RefCoverFont.cover_font_style_css                CSS `font-style` value
                                                   (italic / normal). Loaded by
                                                   the cover renderer when it
                                                   converts the saved
                                                   link_cover_font_id into
                                                   inline style for the preview.
  RefCoverFont.cover_font_weight_css               CSS `font-weight` value
                                                   (300 / 400 / 700). Same
                                                   renderer chain.

================================================================================
ADDING A NEW REF GROUP — checklist
================================================================================
1. SQL: CREATE TABLE [bookwriter].[ref_<name>] with at minimum the columns
   bookwriter_ref_<name>_id (PK), <name>_code (UNIQUE), <name>_name_en, is_active.
2. Seed initial rows.
3. models.py: Add Ref<Name> class with managed=False mapping to the new table.
4. views_api_refs.py: Add one entry to REF_GROUP_TABLE_MAP keyed by the
   group code; supply the (Class, pk, code, name_en, name_bn|None) tuple.
5. (Optional) JS hits /api/refs/<group>/ to populate a dropdown.

================================================================================
WHY THIS MODULE IS STANDALONE
================================================================================
views_api_refs.py imports nothing from the other views_api_* modules and
shares no helpers — it's the cleanest "one feature, one file" split in the
app. Per CLAUDE.md GATE 6 "split files when the feature is atomic and
independent" — refs is the textbook example.
