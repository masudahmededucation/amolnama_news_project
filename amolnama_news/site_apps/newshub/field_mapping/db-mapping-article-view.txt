Article View — Field Mapping
============================
Last updated: 2026-03-24

== URLs ==
Landing: /newshub/articles/   → news_article_landing(request)
Detail:  /newshub/article/<slug>/  → article_detail(request, slug)

== Files ==

== Data Flow ==

1. pub_article (slug lookup)
   [newshub].[pub_article]
   - pub_article_slug → URL slug
   - link_news_entry_id → FK to coll_news_entry
   - pub_article_headline_bn → displayed headline (overrides news entry headline)
   - is_published → must be 1 for public view
   - is_premium → premium badge display
   - published_at → display date

2. coll_news_entry (via pub_article.link_news_entry_id)
   [newshub].[coll_news_entry]
   - news_headline_bn → fallback headline
   - news_summary_bn → summary section
   - news_content_body_bn → body paragraphs (split by newline)
   - occurrence_at → occurrence date in sidenote
   - is_breaking → breaking badge
   - link_form_type_id → determines which investigation table to query
   - link_news_category_id → category badge
   - link_contributor_id → contributor info
   - link_district_id, upazila_city_corporation_name, full_address_bn → location sidenote
   - link_ref_status_article_publication_status_id → publication status badge (818-823)

3. Investigation tables (per form type, via helpers.py)
   Extortion: [investigation].[extortion_form_impact]
   - demand_amount_demanded_bdt → 💰 amount sidenote
   - demand_amount_collected_bdt → 💸 amount sidenote
   - sector_is_* flags → 🏢 sector sidenote
   - accused_is_* flags → 👥 perpetrator affiliation sidenote
   - threat_is_* flags → ⚠️ threat method sidenote
   - consequence_is_* flags → 📋 consequence sidenote

   Extortion Legal: [investigation].[extortion_form_victim_legal_action]
   - link_ref_status_law_gd_fir_status_id → ⚖️ FIR status sidenote
   - gd_fir_case_gd_number → 📝 case number sidenote

4. Actors (shared across all form types)
   [investigation].[incident_involved_actor_profile]
   - incident_involved_actor_role_group_code → ACCUSED/VICTIM/WITNESS
   - link_person_id → FK to person for name lookup
   - actor_organization_name → org name fallback

5. Tags
   [newshub].[coll_news_entry_tag] + [newshub].[ref_news_category_tag]
   - tag_name_bn → tag badges

6. Comments
   [newshub].[eng_comment]
   - link_pub_article_id → FK to pub_article
   - eng_comment_text_bn → comment text
   - is_approved → only approved shown publicly
   - parent_comment_id → threaded replies

7. Community Additions
   [newshub].[article_community_addition]
   - community_addition_type_code → type (correction, additional_info, source_link)
   - community_addition_title_bn → title
   - community_addition_body_bn → body
   - community_addition_source_url → source link
   - status_code → pending/approved/rejected (pending+approved shown, rejected hidden)

8. Article Stats
   [newshub].[eng_article_stat]
   - view_count → display count
   - share_count → share count

== Edit Pre-Population (added 2026-03-24) ==

URL pattern: /newshub/news-collection/multistep/extortion/?edit=<coll_news_entry_id>
Edit button on article detail links to this URL.

Server-side: helpers.py → build_edit_data(entry_id, form_type_code)
  - _build_shared_edit_data(entry) → contributor, news content, location, social sources, tags
  - _build_actor_edit_data(entry_id) → accused/victim/witness from IncidentInvolvedActorProfile + Person
  - _build_extortion_edit_data(entry_id) → ExtortionFormImpact BIT→ID + ExtortionFormVictimLegalAction

BIT flag → status_id reverse maps (helpers.py):
  _EXTORTION_SECTOR_BIT_TO_ID:       sector_is_* → 427-436
  _EXTORTION_AFFILIATION_BIT_TO_ID:  accused_is_* → 444-451
  _EXTORTION_THREAT_BIT_TO_ID:       threat_is_* → 452-460
  _EXTORTION_CONSEQUENCE_BIT_TO_ID:  consequence_is_* → 461-470
  _EXTORTION_CONTEXT_BIT_TO_ID:      context_is_* → 472-473
  Legal BIT→code maps resolved via RefStatus queries.

Client-side: news-form-edit-load.js (loads BEFORE component scripts)
  Phase 1 (immediate): Sets hidden JSON inputs (#accused-json, #victim-json, #witness-json,
    #extortion-incident-json, #ext-legal, #social-source-json) so components find data on init.
  Phase 2 (deferred, after window.load): Cascade fields (district, category, tags via
    window.newshubTags.add() API).

news-form-persist.js: Skips localStorage restore when window.__EDIT_MODE__ is true.

Submit (POST): _handle_news_submission() detects edit_entry_id in POST data.
  - UPDATE existing CollNewsEntry + CollContributor (not CREATE)
  - DELETE + re-INSERT junction records (tags, actors, social sources, form-specific)
  - Duplicate headline check excludes own entry
  - Redirect to article view page after save (via PubArticle slug lookup)

== Form Type Badge (added 2026-03-24) ==

Form type (e.g. চাঁদাবাজি) shown as badge on:
  - Article detail meta bar: .article-meta-form-type (purple #6d5e9b)
  - Article landing cards: .article-landing-card-form-type (purple #6d5e9b)
Source: RefNewsFormType.form_name_bn via form_type context variable.

== Status Widget (added 2026-03-24) ==

Single element with two states:
  Default: Badge (.article-status-badge) + pencil Edit button (.article-status-edit-btn)
  Click Edit: Badge hides, dropdown appears (.article-status-select)
  Select new status: AJAX POST to /newshub/api/article/<id>/status/, updates badge, reverts to badge mode
  Click away: Reverts to badge mode

Permission: Admin, staff, OR article owner can change status.
All feedback via inline messages (no popups — see feedback_no_popups.md).

== Rich Text Editor (added 2026-03-24) ==

Quill.js 1.3.7 via CDN (Snow theme) on summary_bn + content_body_bn fields.
Loaded in: news-multistep-base.html + news-collection.html (covers all 13+ forms).
Init: news-quill-init.js → window.__quillNewsSummary, window.__quillNewsBody
Sync: auto on text-change + explicit on form submit.
Sanitization: _sanitize_rich_html() in views.py strips script/style/iframe/on*/javascript: URLs.
Article detail renders body paragraphs with |safe (block <p> tags stripped, inline HTML preserved).

== Permissions ==

View: Public (no login)
Edit own: Login required (privilege level 1+)
Suggest edits: Login required (privilege level 2, 50+ points)
Edit others: Login required (privilege level 3, 200+ points)
Approve: Login required (privilege level 4, 500+ points)
Delete: Admin only (privilege level 5)
