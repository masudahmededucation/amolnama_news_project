  LANGUAGE TOGGLE WIRING — PROGRESS TRACKER
  Feature: news-form-lang.js — Bengali/English toggle in header
  Last updated: 2026-03-06
================================================================================

HOW IT WORKS
────────────
1. Header radio buttons: input[name="form_lang"] value="bn" or "en"
2. news-form-lang.js scans DOM for [data-bn][data-en] and [data-ph-bn][data-ph-en]
3. On toggle: swaps textContent of labels/headings; swaps placeholder of inputs
4. Saves preference via AJAX POST to /account/api/language-pref/
5. Next page load: UserProfile.language_pref drives which radio is checked

RULES FOR WIRING TEMPLATES
────────────────────────────
• Regular <label>, <h3>, <h4>, <p> elements:
    <label data-bn="Bengali text" data-en="English text">Bengali text</label>
    NOTE: set textContent to Bengali initially (data-bn = displayed default)

• <input> and <textarea> placeholders:
    data-ph-bn="বাংলা প্লেসহোল্ডার..." data-ph-en="English placeholder..."

• <label class="checkbox-inline"> or <label class="radio-inline"> wrapping <input>:
    DO NOT put data-bn/data-en on the label itself — JS clears textContent which
    re-orders the input after the text. Instead, wrap only the text in a <span>:
    <label class="checkbox-inline">
      <input type="checkbox" ...>
      <span data-bn="বাংলা" data-en="English">বাংলা</span>
    </label>

• <option value=""> default (select placeholder):
    <option value="" data-bn="-- নির্বাচন করুন --" data-en="-- Select --">-- নির্বাচন করুন --</option>

• Tom Select inputs: placeholder is set by Tom Select JS, not by the HTML attribute.
    data-ph-bn/data-ph-en do NOT work on Tom Select-enhanced inputs.

• JS-generated content (repeater cards, DB-driven checkboxes/radios):
    NOT covered by template wiring. JS would need to call
    window.newshubFormLang.current() when rendering to apply the current language.

================================================================================
  KNOWN LIMITATIONS (all forms)
================================================================================

1. JS-GENERATED CONTENT NOT WIRED:
   All DB-driven radio/checkbox/select options are rendered by JS (news-wcv-*.js etc.)
   and always use Bengali labels. The language toggle cannot affect these after
   page load without JS changes.
   Fix needed: each component JS must check window.newshubFormLang.current()
   when populating checkboxes/radios, and listen for a custom event when the
   language changes.

2. TOM SELECT PLACEHOLDERS NOT WIRED:
   Tom Select replaces the native <input> placeholder with its own. The
   data-ph-bn/data-ph-en approach does not work for Tom Select controls.
   Affected: police station search (#wcv-police-station), location search,
   district/category dropdowns.

3. REPEATER CARD FIELDS NOT WIRED (Step 6 accused cards):
   All fields in wcv-accused-form.html card repeater are JS-generated.
   Partial fix wired (section title + add button). Full fix requires JS changes.
