SAVE SEQUENCE (on form submit — in _handle_news_submission)
================================================================================
1. Read 2 plain POST keys: entertainment_medium_id, entertainment_sub_type (status_code strings)
2. Parse 3 hidden JSON inputs: entertainment_production_json,
   entertainment_cast_release_json, entertainment_performance_json
3. Resolve FK values:
   - link_ref_status_entertainment_form_medium_type_id = _ref_id('entertainment_form_medium_type', entertainment_medium_id)
   - link_ref_status_entertainment_form_issue_sub_type_id = _ref_id('entertainment_form_issue_sub_type', entertainment_sub_type)
   - prod_link_ref_language_id = _ref_id('entertainment_form_language', status_code)
   - prod_link_ref_entertainment_industry_id = _ref_id('entertainment_form_entertainment_industry', status_code)
   - cast_link_ref_media_platform_id = _ref_id('entertainment_form_media_platform', status_code)
   - cast_link_ref_genre_category_id = _ref_id('entertainment_form_genre_category', status_code)
   - perf_link_ref_audience_response_id = _ref_id('entertainment_form_audience_response', status_code)
4. EntertainmentFormFact.objects.create() — single row with all 18 data columns
   prod_title_name_bn is NOT NULL — fallback to empty string if empty
   created_at = now (Python-supplied, not DB default)
