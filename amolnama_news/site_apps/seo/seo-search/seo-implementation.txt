================================================================================
WHAT GETS RENDERED ON EVERY PAGE (via base.html)
================================================================================

1. <meta name="description">      — Dynamic per-page description
2. <link rel="canonical">          — Canonical URL (prevents duplicate content)
3. <meta property="og:*">          — Open Graph tags (Facebook, LinkedIn sharing)
4. <meta name="twitter:*">         — Twitter Card tags
5. <meta name="robots">            — noindex/nofollow for private pages
6. <meta name="theme-color">       — Browser theme color
7. <link rel="icon">               — Favicon
8. <link rel="apple-touch-icon">   — iOS home screen icon
9. JSON-LD Organization schema     — Tells Google who we are
10. JSON-LD WebSite schema         — Enables sitelinks search box
11. JSON-LD BreadcrumbList         — Breadcrumb navigation in search results
12. JSON-LD NewsArticle            — Rich snippets for articles (headline, author, date, image)

================================================================================
HOW TO ADD SEO TO A NEW VIEW
================================================================================

In any view, add an 'seo' dict to the context:

    def my_view(request):
        context = {
            "seo": {
                "title": "Page Title — আমলনামা নিউজ",
                "description": "Bengali description. English description.",
                "og_image": request.build_absolute_uri("/path/to/image.jpg"),
                "og_type": "website",           # or "article"
                "noindex": False,                # True for private pages
                "breadcrumbs": [
                    {"name": "হোম", "url": "/"},
                    {"name": "Section", "url": "/section/"},
                ],
                "json_ld": { ... },              # Optional custom schema
            },
        }
        return render(request, "template.html", context)

If no 'seo' dict is provided, defaults from context_processors.py are used.

================================================================================
ROBOTS.TXT (/robots.txt)
================================================================================

Allows all crawlers by default.
Disallows:
  - /admin/          (Django admin)
  - /account/        (user auth pages)
  - /accounts/       (allauth)
  - /api/            (API endpoints)
  - /newshub/        (internal data collection forms)
  - All /*/api/ paths within apps
Points to sitemap.xml and llms.txt.

================================================================================
SITEMAP.XML (/sitemap.xml)
================================================================================

Generated by Django's contrib.sitemaps framework.
Registered sitemap classes:
  - StaticPageSitemap  — home, about, contact, communityvoice
  - ArticleSitemap     — all articles (priority 0.9, daily)
  - ToolsSitemap       — all 8 tool pages (priority 0.7, monthly)
  - VotingSitemap      — evaluation + election vote (priority 0.6)
  - MarriageSitemap    — marriage service pages (priority 0.5)
  - PoemSitemap        — poetry section (priority 0.4)

To add new pages to sitemap:
  1. Create a new Sitemap class in sitemaps.py
  2. Add it to the SITEMAPS dict at the bottom
  3. That's it — urls.py picks it up automatically

================================================================================
LLMS.TXT (/llms.txt) — AI CRAWLER GUIDE
================================================================================

A 2025/2026 standard for AI crawlers (GPT, Gemini, Perplexity, Claude).
Describes what the site is about, lists sections and tools.
AI models use this to cite and recommend the site to users.

================================================================================
NOINDEX PAGES (not crawled by search engines)
================================================================================

These pages have <meta name="robots" content="noindex, nofollow">:
  - All /account/* pages (login, signup, profile, password reset)
  - All /newshub/* pages (internal data collection forms)
  - Article not found pages
  - sidebar_past_vote_results (partial template)

================================================================================
JSON-LD SCHEMAS
================================================================================

1. NewsMediaOrganization (every page)
   - name, alternateName, url, logo, description

2. WebSite (every page)
   - name, alternateName, url

3. BreadcrumbList (pages with breadcrumbs)
   - Hierarchical navigation path

4. NewsArticle (article detail pages)
   - headline, author, publisher, datePublished, image, timeRequired

================================================================================
SEO BEST PRACTICES APPLIED
================================================================================

1. Every page has unique title + description (Bengali primary, English secondary)
2. Every page has canonical URL (prevents duplicate content penalties)
3. Every page has Open Graph + Twitter Card meta (social sharing previews)
4. Article pages have full NewsArticle JSON-LD (rich snippets in Google)
5. Organization + WebSite schemas on every page (knowledge panel)
6. Breadcrumb schemas for hierarchical navigation in search results
7. Private pages excluded from indexing (noindex + robots.txt)
8. Sitemap covers all public pages with correct priorities
9. llms.txt for AI search engines (Perplexity, Gemini, etc.)
10. Hero images have loading="eager" + fetchpriority="high" for LCP
11. Images have proper alt text from article data
12. html lang="bn" set correctly for Bengali content
