Travel Hub — Field Mapping & Feature Documentation
=====================================================
Last updated: 2026-03-25

== URLs (prefix: /bangladesh-tourist-destinations/) ==
Landing:        /bangladesh-tourist-destinations/                          → bangladesh_landing(request)
Travel Hub:     /bangladesh-tourist-destinations/travel/                   → travel_hub(request)
Add:            /bangladesh-tourist-destinations/travel/add/               → travel_hub_add(request) [login required]
Edit:           /bangladesh-tourist-destinations/travel/add/?edit=<id>     → travel_hub_add(request) [same view, edit mode]
Detail (slug):  /bangladesh-tourist-destinations/travel/<slug>/            → travel_hub_detail_by_slug(request, destination_slug)
Detail (ID):    /bangladesh-tourist-destinations/travel/id/<id>/           → travel_hub_detail_by_id → 301 redirect to slug URL
Beauty Hub:     /bangladesh-tourist-destinations/beauty/                   → beauty_hub(request)

Old URLs (/bangladesh/*) → 301 redirect to /bangladesh-tourist-destinations/* via project urls.py

API — Destinations:
  GET  /bangladesh/api/destinations/                       → api_destination_list (paginated)
  POST /bangladesh/api/destinations/create/                → api_destination_create
  POST /bangladesh/api/destinations/<id>/update/           → api_destination_update

API — Community Contributions (add):
  POST   /bangladesh/api/destination/<id>/photo/             → api_destination_photo_upload (multipart)
  POST   /bangladesh/api/destination/<id>/youtube/           → api_destination_youtube_link_add (JSON)
  POST   /bangladesh/api/destination/<id>/link/              → api_destination_reference_link_add (JSON)

API — Community Contributions (edit/delete):
  PATCH  /bangladesh/api/destination/<id>/photo/<photo_id>/           → api_destination_photo_update (JSON: caption_bn)
  DELETE /bangladesh/api/destination/<id>/photo/<photo_id>/delete/    → api_destination_photo_delete
  PATCH  /bangladesh/api/destination/<id>/youtube/<yt_id>/            → api_destination_youtube_link_update (JSON: video_title_bn, description_bn)
  DELETE /bangladesh/api/destination/<id>/youtube/<yt_id>/delete/     → api_destination_youtube_link_delete
  PATCH  /bangladesh/api/destination/<id>/link/<ref_id>/              → api_destination_reference_link_update (JSON: reference_title_bn, description_bn)
  DELETE /bangladesh/api/destination/<id>/link/<ref_id>/delete/       → api_destination_reference_link_delete

== Rich Text Editor (Quill.js) ==

Quill.js 1.3.7 via CDN (Snow theme).
Applied to: Short description + Detailed description fields (travel hub).
Also applied to: Beauty Hub upload (description), Newshub all forms (summary + body).

Toolbar: Bold, Italic, Underline | H2, H3 | Text Color, Background Color | Ordered/Bullet List | Blockquote | Link | Clean

quill-editor.js exposes: window.initQuillEditor(containerId, hiddenTextareaId, opts)
  Returns: { quill, syncToHidden(), setContent(html) }

HTML stored in DB columns (NVARCHAR). Sanitized on save via _sanitize_html() in views_api.py.
  Strips: script, style, iframe, object, embed, form tags
  Strips: on* event attributes
  Strips: javascript: URLs

Display: paragraphs split by \n, block <p> tags removed, inline formatting preserved via |safe.
Card previews: |striptags|truncatewords:25 (plain text only).

CSP updated: cdn.quilljs.com added to script-src and style-src in core/base.html.

== Edit Pre-Population ==

URL: /bangladesh/travel/add/?edit=<destination_id>
View detects ?edit=<id>, fetches CollDestination, serializes to JSON.
JS reads JSON from #th-edit-data, populates form fields + Quill editors.
Submit POST goes to /bangladesh/api/destinations/<id>/update/ (not create).
Cancel button links back to detail page.

All bangladesh pages: main a, main a:hover, main a:focus { text-decoration: none; }

== YouTube Video Display ==

Thumbnail from i.ytimg.com (CSP-allowed) + red play button overlay.
Click opens YouTube in new tab (works for ALL videos, no embed restriction issues).
Video ID extracted from URL via regex in _extract_youtube_video_id().
Supports: youtube.com/watch?v=, youtu.be/, youtube.com/embed/, youtube.com/shorts/

== Photo Upload ==

Multipart POST to /bangladesh/api/destination/<id>/photo/
Saves to /media/upload/bangladesh/destination-photo/<uuid>.ext
Allowed: JPEG, PNG, WebP
Creates CollDestinationPhoto row with caption, sort_order, is_cover=False.

_sanitize_html(html) in views_api.py:
  - Removes script/style/iframe/object/embed/form tags and content
  - Removes on* event handler attributes
  - Replaces javascript: URLs with #
  Applied to: destination_description_bn, destination_short_description_bn
  in both api_destination_create and api_destination_update.

== Community Contribution Permissions ==

Permission matrix for community contributions (photos, YouTube links, reference links):
  Add:         Any authenticated user can upload/add to any destination
  Edit:        Contribution owner + destination owner + staff/admin
  Delete:      Contribution owner + destination owner + staff/admin
  Edit main:   Destination owner + staff/admin only (can_edit flag)

Permission check function: _can_manage_contribution() in views_api.py
Template: can_manage flag annotated per contribution in views.py travel_hub_detail()

Uploader attribution: display_name from UserProfile, bulk-fetched to avoid N+1.
Edit: inline form appears inside the contribution card (photo/youtube/link).
Delete: inline confirmation bar (red) — no popups.

== Media Card Layout (Photos + Videos) ==

Photos and videos use unified card layout (not background-image thumbnails):
  Image at top → caption → footer bar (like, views, uploader, time ago, edit/delete)

DB tables:
  eng_destination_photo_like — photo like toggle (unique per user+photo)
  eng_destination_video_like — video like toggle (unique per user+video)
  coll_destination_photo.view_count, like_count — counters
  coll_destination_youtube_link.view_count, like_count — counters
  coll_destination_youtube_link.video_platform — 'youtube', 'tiktok', 'instagram', etc.
  coll_destination_youtube_link.video_thumbnail_url — fetched from oEmbed (TikTok) or i.ytimg.com

API endpoints:
  POST .../photo/<id>/view/   — increment view (fires on lightbox open)
  POST .../photo/<id>/like/   — toggle like
  POST .../video/<id>/view/   — increment view (fires on video click)
  POST .../video/<id>/like/   — toggle like
  POST .../destination/<id>/review/ — add review (1 per user per destination)

Multi-platform video support:
  YouTube: auto-thumbnail from i.ytimg.com, clean URL stored
  TikTok: thumbnail fetched via oEmbed API server-side
  Instagram/Facebook: placeholder icon
  Platform auto-detected from URL
