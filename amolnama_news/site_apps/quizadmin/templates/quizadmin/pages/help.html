A guided tour of Quiz Panel — what each tab does, in the order you'll use them.

The full workflow, left to right

Every box is a tab in the top nav. Arrows show what feeds what. Click a box to jump there.

  1. 1 Books Upload PDFs of your source material — textbooks, constitutions, scripture, exam guides. Action: Upload PDF
  2. 2 AI Jobs Trigger Ollama to generate questions from each book chapter, with NLI anti-hallucination. Action: Start generation
  3. 3 Review Approve or reject AI-generated questions one by one, with NLI confidence visible. Action: Approve / reject
  4. 4 Questions Browse the universal question bank. Filter, search, edit, version, or hand-author new ones. Action: Edit / author
  5. 5 + Create Quiz Pick questions from the bank, set scoring, time limit, proctoring level, certificate template. Action: Assemble + publish
  6. 6 Quizzes Manage every quiz — clone, edit, schedule, archive, leaderboard, certificate template. Action: Manage lifecycle
  7. 7 Analytics Visual charts: score distribution, pass rate, hardest questions, topic engagement. Action: Measure outcomes
  8. 8 Proctoring Live feed of integrity violations across all sessions. Forgive false positives. Action: Audit + forgive
  9. 9 Creators Grant non-staff users permission to create quizzes; set expiry dates, revoke at any time. Action: Grant / revoke

Who does what

Staff (admin)

  • Upload books and start AI generation jobs.
  • Approve / reject AI-generated questions in the Review queue.
  • Create quizzes; publish, archive, or delete them.
  • Set the proctoring level and certificate template per quiz.
  • Forgive proctoring false positives.
  • Grant quiz-creator rights to non-staff users.

Quiz creator (non-staff with permission)

  • Receive a permission grant from staff (visible in their messenger inbox).
  • Author new quizzes — but only edit ones they themselves created.
  • See analytics for their own quizzes.

Student (any logged-in user)

  • Take any published quiz from the consumer site (historybd, constitutionbd, etc.).
  • If the quiz is level 2 (webcam), see the privacy consent screen first.
  • Get score + correct-answer review immediately on submit.
  • Receive a pulse notification + DM when results are ready.
  • Earn streaks, badges, mastery, and a public-verifiable certificate (if the quiz issues one).

Frequently asked questions

How do I create my first quiz?

1. Click + Create Quiz.

2. Fill in the title, topic, time limit, pass percentage, and proctoring level (defaults to 1 — browser lockdown).

3. Either pick existing questions from the bank, or hand-author new ones inline.

4. Save as draft, then publish from the Quizzes list when ready.

How do AI-generated questions work?

Upload a PDF in Books. The system extracts text via PyMuPDF (with Tesseract OCR fallback for scanned pages), splits into chunks, and stores them.

Then in AI Jobs you trigger Ollama (local LLM) to generate N questions per chunk. Every generated question is fed through an NLI (Natural Language Inference) classifier — if the proposed answer doesn't entail from the source chunk, the question is auto-rejected as a hallucination.

Surviving questions land in the Review queue with their NLI confidence visible. You approve / reject before they enter the question bank.

What are the 3 proctoring levels?

Level 0 — None. No anti-cheating. Use for casual practice quizzes.

Level 1 — Browser lockdown (default). Detects tab switching, copy/paste attempts, fullscreen exit, dev-tools shortcuts. No camera. No consent needed.

Level 2 — Lockdown + webcam AI (opt-in only). Adds face presence + multi-face + gaze tracking via MediaPipe in the browser. ZERO IMAGES — nothing leaves the browser; only text events ("look_away", "multiple_faces") with timestamps. Requires student consent screen before the quiz starts.

How do certificates work?

For each quiz, click Certificate in the row actions on the Quizzes page. Paste an HTML template (or click Load default for a polished bilingual one). Save.

Now whenever a student passes that quiz, the engine auto-issues a certificate with an unforgeable serial. The student gets a public verify URL: /mastermind/certificate/<serial>/. Anyone can verify authenticity.

The student saves it as PDF via Ctrl+P (browser-native, no server PDF library).

How do students get notified when results are ready?

Two channels, both in-house, both free:

1. A pulse notification badge on their homepage (newsengine.NotificationItem + WebSocket).

2. A direct message in their messenger inbox from a designated system sender account (configure with the MASTERMIND_SYSTEM_SENDER_PROFILE_ID env var; without it the DM is silently skipped and only pulse fires).

No email. No SMTP. No third party.

Can I let a non-staff user create quizzes?

Yes — go to Creators, search by email, set an optional expiry date, click Grant. The user gets a notification and can immediately access Quiz Panel. They can edit only the quizzes they themselves created. Staff can still edit anything.

A student needs extra time (accommodation). How do I grant it?

Once the student starts a session, you can POST to /mastermind/api/session/<session_id>/accommodation/ with { "extra_time_minutes": 15, "notes": "documented dyslexia" } or { "no_time_limit": true }. The engine honors the override on the next page reload. (UI for this lands on the proctoring session audit page in a future round.)

How do I plug Mastermind into Slack / Discord / Zapier?

POST to /mastermind/api/webhook/subscriptions/create/ with { "webhook_event_code": "quiz_session_passed", "webhook_target_url": "https://...", "webhook_secret": "..." }. Whenever the event fires, Mastermind POSTs a signed JSON envelope to your URL. The signature is HMAC-SHA256 in the X-Mastermind-Signature header — verify it on your side to guarantee authenticity.

Supported event codes: quiz_session_completed / passed / failed, ai_generation_completed, quiz_creator_permission_granted, proctoring_violation_logged.

Can I export questions to back up or migrate?

Yes. GET /mastermind/api/question/export/?format=csv exports the entire bank as a CSV (UTF-8 with BOM so Excel opens Bengali correctly). ?format=json exports as JSON. Both are round-trip compatible with the bulk-import endpoint at /mastermind/api/question/bulk-import/.

For a single quiz with its question pool: GET /mastermind/api/exam/<id>/export/.

What does "ZERO IMAGES" actually mean in proctoring?

Three guarantees:

  1. Schema-level: the coll_quiz_proctoring_log table has no BLOB / VARBINARY column. Even if rogue code wanted to store an image, it physically cannot.
  2. Client-level: all face / landmark / gaze processing happens in WebAssembly + WebGL inside the browser. The MediaPipe stream attaches to a hidden in-page <video>, never sent over the network.
  3. Network-level: our CSP only allows POSTs to 'self'. Even a hostile script cannot exfiltrate frames to a third party.

The server only ever receives small text events: { type: "look_away", timestamp: "2026-04-17T10:23:11Z" }.

What question types are supported?

Eight types, ordered by difficulty to author:

  1. MCQ single — pick one correct option.
  2. MCQ multi — pick all correct options.
  3. True / False — binary.
  4. Fill in the blank — short text answer matched against allowed answers.
  5. Short answer — free text, requires manual grading.
  6. Essay (paragraph) — long-form, requires manual grading.
  7. Matching — pair stems with responses.
  8. Ordering / sequence — arrange items into the correct order.
