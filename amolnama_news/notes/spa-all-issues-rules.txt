ISSUE 1: DOUBLE FLASH DURING SPA NAVIGATION
================================================================

ROOT CAUSE (verified by research):
- The first-load cloak adds class html.spa-ready on DOMContentLoaded
- html.spa-ready is NEVER removed after that
- CSS rule: html.spa-ready main { opacity: 1 } — specificity 0,1,0,1
- SPA sets mainElement.style.opacity = '0' — inline style wins (correct)
- BUT: inline critical CSS main { opacity: 0 } has NO transition: none
- When base.css loads, it applies transition: opacity .15s ease to main
- This can cause the initial reveal to ANIMATE instead of being instant
- Also: base.css transition rule may briefly conflict with JS transition='none'

RULE:
- Inline critical CSS must include transition: none on main
- The spa-ready class approach is WRONG for SPA — it fights with SPA navigation
- REDESIGN: Remove spa-ready class entirely. Use visibility: hidden (not opacity)
  in inline CSS. SPA controls opacity via inline styles. First load reveals via
  a one-time inline script that sets visibility: visible after DOMContentLoaded.
  visibility: hidden prevents ALL rendering (no layout, no flash). opacity: 0
  still paints (invisible but elements take space and can flash sub-elements).

GOLD STANDARD APPROACH:
- Inline CSS: main { visibility: hidden }
- DOMContentLoaded: mainElement.style.visibility = 'visible'
- SPA navigation: uses opacity for smooth fade (transition on reveal only)
- No CSS class toggling. No specificity fights. Inline styles always win.

ISSUE 2: SPA SCRIPT LOADING — CDN SCRIPTS FAIL
================================================================

ROOT CAUSE (verified by research):
- loadPageJs() appends ALL scripts to DOM without waiting
- External CDN scripts (Quill, Flatpickr) load async
- Dependent local scripts execute immediately before CDN finishes
- quill-editor.js line 40: new Quill(...) — NO guard check, crashes
- news-quill-init.js: has guard for initQuillEditor but not for Quill itself
- Tom Select and Leaflet have guard checks (typeof TomSelect/L === 'undefined')

RULE:
- loadPageJs() must wait for external scripts to load before executing
  dependent scripts. Same Promise pattern as loadPageCss().
- ALL scripts that depend on CDN libraries MUST have guard checks:
  if (typeof Quill === 'undefined') return;
- Script loading must preserve execution order: externals first, locals second.

GOLD STANDARD APPROACH:
- Split scripts into two groups: external (src with CDN URLs) and local
- Load external scripts first, wait for all onload events via Promise.all
- Then load local scripts in order
- Also add typeof guard to quill-editor.js as defensive measure

ISSUE 3: display-hidden CLASS FLASHES BEFORE CSS LOADS
================================================================

ROOT CAUSE (verified by research):
- .display-hidden { display: none } defined in forms.css (line 73)
- 355 HTML elements use class="display-hidden"
- 373 JS instances toggle this class across 192 files
- BEFORE forms.css loads, the class does nothing — elements are VISIBLE
- Overlays, modals, hidden panels all flash on page load

RULE:
- Use HTML hidden attribute, NOT CSS class, for initial hiding
- CSS rules with explicit display (flex, grid, block) OVERRIDE hidden attribute
- Every element with display: flex/grid/block MUST use :not([hidden]) pattern:
  .element:not([hidden]) { display: flex; }
- photo-lightbox.css already demonstrates correct pattern (line 15-17)

COMPLEXITY:
- 355 HTML template changes (class="display-hidden" -> hidden attribute)
- 373 JS changes (classList.add/remove/toggle/contains -> .hidden property)
- CSS audit: find all elements that use display-hidden AND have explicit display rules
- These elements need :not([hidden]) added to their CSS

MIGRATION PATTERNS:
  HTML:  class="display-hidden"           -> hidden
  HTML:  class="foo display-hidden"       -> class="foo" hidden
  JS:    classList.add('display-hidden')   -> element.hidden = true
  JS:    classList.remove('display-hidden') -> element.hidden = false
  JS:    classList.toggle('display-hidden') -> element.hidden = !element.hidden
  JS:    classList.toggle('display-hidden', bool) -> element.hidden = bool
  JS:    classList.contains('display-hidden') -> element.hidden
  CSS:   .foo { display: flex }           -> .foo:not([hidden]) { display: flex }
         (only if .foo elements also use hidden attribute)
