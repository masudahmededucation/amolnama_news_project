Ask for full permission to edit/write ALL files at the start of a task, not one file at a time.

**Why:** The user has to click approve for every single file edit. When there are 10+ files to change, this is extremely annoying and wastes time. Get blanket approval upfront.

**How to apply:**
- At the start of a multi-file task, tell the user: "I need to edit X files — proceeding now"
- Never ask "can I edit this file?" for each individual file
- The user has already given auto-accept mode — proceed with changes without per-file confirmation
