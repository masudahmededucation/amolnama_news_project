## Messenger App (WhatsApp-style)

### DB Tables (5 tables in [messenger] schema)
- conversation — type (direct/group), denormalized last_message fields
- conversation_participant — unread_count, is_pinned, is_muted, last_read_message_id
- message — text, status (sent/delivered/read), reply_to, edit, delete_for_everyone
- message_deletion — per-user "delete for me"
- typing_indicator — ephemeral, 5-second expiry

### API Endpoints (14 total)
- GET /api/unread-count/ — total unread (polled by sidebar nav)
- GET /api/conversations/ — list with sorting (pinned first, then by time)
- POST /api/conversations/start/ — start or find existing direct chat
- GET /api/messages/<id>/ — paginated messages (cursor-based, ?before=)
- POST /api/messages/<id>/send/ — send with optimistic UI
- GET /api/messages/<id>/poll/ — new messages since ?after=
- POST /api/messages/<id>/read/ — mark all as read
- POST /api/messages/<id>/delete-for-me/<mid>/ — soft delete for current user
- POST /api/messages/<id>/delete-for-everyone/<mid>/ — delete for all (1 hour limit)
- POST /api/messages/<id>/edit/<mid>/ — edit message (15 min limit)
- POST /api/conversations/<id>/pin/ — toggle pin
- POST /api/conversations/<id>/mute/ — toggle mute
- POST /api/typing/<id>/ — update typing indicator
- GET /api/typing/<id>/status/ — check if other user typing

### Features Built
- Two-panel layout (280px sidebar + chat)
- 3-second polling for new messages
- 30-second polling for conversation list
- Optimistic message sending (instant display)
- Read receipts (✓ sent, ✓✓ delivered, ✓✓ blue read)
- Reply-to with quoted preview (click to scroll to original)
- Context menu (right-click desktop, long-press mobile): reply, copy, edit, delete
- Edit message (inline contentEditable, 15 min limit)
- Delete for me / delete for everyone (1 hour limit)
- Typing indicator display ("টাইপ করছেন...")
- Scroll up to load older messages (cursor pagination, preserves scroll)
- Conversation search (client-side filter)
- Emoji picker (50 emojis, inside textarea)
- Pin/mute conversation
- Unread badge on sidebar nav (red circle, polls 30s)
- Message button on user profiles (/messenger/?start=<id>)
- Date+day+time on every message (DD/MM/YYYY বারের_নাম HH:MM AM/PM)
- Sender name + avatar on received bubbles
- Date separators with gap between groups
- Send failure: message stays with retry button (persists in localStorage)
- Mobile: single-panel mode with back button, 44px tap targets
- Block check prevents messaging
- Notification on new message
