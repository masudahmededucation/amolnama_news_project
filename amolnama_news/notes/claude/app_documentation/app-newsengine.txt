================================================================================
APP: NEWSENGINE — Feed Brain, Ranking, Personalization, Notifications
================================================================================

URL Prefix: /newsengine/
App Name: newsengine
Role: The brain of the platform — decides what content users see on the home feed.

================================================================================
ARCHITECTURE
================================================================================

Core → Newsengine → Pulse
- core/views.py calls newsengine/feed_builder.py
- feed_builder assembles the complete feed
- pulse template renders whatever feed_builder provides

================================================================================
FILES
================================================================================

feed_builder.py     — Assembles home feed: posts + promos + boosts + ranking + dedup + filtering
promo_builders.py   — Collects published content from all 7 apps + tools catalog
ranking.py          — Content scoring: recency(0.3) + engagement(0.4) + reputation(0.1) + quality(0.2)
trending.py         — Finds fastest-growing content in last 24h
personalization.py  — User interest profile from view history, boosts matching content
recommendations.py  — "You liked X, you might like Y" — category-based collaborative filtering
notifications.py    — Global notification helper: create_notification(), get_unread_count(), mark_all_read()
rate_limiter.py     — Per-action rate limits: check_rate_limit(user_id, action_code)
models.py           — FactUserContentView, FactContentScore, CollNotification, CollContentBookmark, FactRateLimitLog, CollHashtag, CollPostHashtag
views_api.py        — APIs: notifications, bookmarks, feed pagination, recommendations, trending hashtags
urls.py             — 5 API routes

================================================================================
FEED PIPELINE (feed_builder._build_intelligent_feed)
================================================================================

1. Inject promo cards (published items at top + random boost lower)
2. Apply content ranking (sort posts by score, promos keep position)
3. Apply personalization boost (user interest profile)
4. Apply user feed preferences (hide categories user turned off)
5. Remove viewed content (read history, last 200)
6. Exclude blocked users
7. Deduplicate (same content as published + boost)
8. Category filter (if ?category=poem param)

================================================================================
RANKING FORMULA
================================================================================

total_score = (recency × 0.3) + (engagement × 0.4) + (author_reputation × 0.1) + (quality × 0.2)

- Recency: exponential decay, half-life 48 hours
- Engagement: log-normalized (likes + views×0.1 + replies×2 + reposts×1.5 + votes)
- Author reputation: log-normalized contribution_score_count
- Quality: word count + vocabulary diversity

================================================================================
PROMO SYSTEM (promo_builders.py)
================================================================================

3 types of feed appearance:
1. Published — all published items at top, sorted by date
2. Random boost — 2 random per category scattered lower in feed
3. Tools — 2 random tools promoted per page load

Apps covered: Debate, News, Poem, Story, Art, Travel, Tools (10 tools)

================================================================================
CONTENT CLASSIFICATION ENGINE (2026-04-02)
================================================================================

content_classifier.py — keyword-based auto-categorisation of posts and arguments.

classify_text(text):
- NFC normalize Bengali text
- Match against ref_flagged_keyword per category (keyword × weight)
- Pattern matching: phone numbers, URLs, money, repeated chars, excessive emojis
- Returns list of (category_code, score, action) sorted by score
- Threshold: score > 0.5 = auto-flag for harmful categories

classify_and_store(content_source_app, content_id, text):
- Classifies text
- Stores audit in fact_content_classification
- Updates post: content_category_code, is_auto_flagged
- If action = 'auto_hide' and score > 0.5: sets is_published = False

Hooked into:
- post/views_api.py api_post_create() — background thread
- debate/views_api.py api_post_argument() — background thread

18 Categories:
- Allow + label: safe, religion, politics, education, business, news, entertainment
- Auto-hide + flag: spam, adult, violence, hate_speech, harassment, scam, self_harm, drugs
- Label only: misinformation, copyright, advertisement

DB Tables:
- [newsengine].[ref_content_category] — 18 seed categories with action + severity
- [newsengine].[ref_flagged_keyword] — Bengali + English keywords per category
- [newsengine].[fact_content_classification] — audit trail

Link Preview API (shared):
- /newsengine/api/link-preview/?url=... — fetches og:title, og:description, og:image
- Used by post + debate (moved from debate to newsengine for zero duplication)

## New Features (2026-04-06) — Feed Cache + Embeddings + Video Transcoding

### Feed Score Caching (feed_cache.py)
- Populates [newsengine].[fact_feed_content_score] with pre-calculated ranking scores
- MERGE (upsert) pattern — insert on post create, update on engagement change
- Triggered from: api_post_create, api_post_like_toggle, api_post_vote_toggle, api_post_repost (all via _refresh_post_feed_score_background)
- Scores: engagement, trending, recency, total (same formula as ranking.py)
- refresh_stale_scores() for periodic background refresh of old scores
- get_cached_scores() for batch read of cached scores
- Replaces per-request recalculation with cached values

### Content Embeddings (embeddings.py)
- sentence-transformers model: paraphrase-multilingual-MiniLM-L12-v2 (384-dim, supports Bengali+English)
- Lazy-loaded singleton — thread-safe, first load ~500MB, then ~150MB RAM
- encode_and_store_embedding() — encode text + upsert into [newsengine].[fact_content_embedding]
- encode_and_store_embedding_background() — non-blocking background thread version
- find_similar_content() — cosine similarity via numpy (same content type)
- find_similar_content_cross_type() — cross-app semantic search
- Wired into post create (background thread after fact-check)

### Enhanced Recommendations (recommendations.py)
- get_recommendations_for_user_enhanced() — vector similarity + category fallback
- Strategy: find user's most-liked posts → get semantically similar content → fill with category-based
- get_similar_posts() — individual post similarity lookup
- API endpoint /newsengine/api/recommendations/ now uses enhanced version

### Video Transcoding (post/video_transcoder.py)
- FFmpeg HLS adaptive bitrate conversion in background thread
- queue_video_transcode() called after video upload in api_post_create
- Creates 360p (800kbps) and 720p (2500kbps) quality variants
- Generates HLS master manifest (.m3u8) for adaptive streaming
- Job tracking in [media].[fact_video_transcode_job] (pending → processing → completed/failed)
- get_hls_manifest_url() for retrieving completed transcode output
- Requires: FFmpeg binary installed (winget install ffmpeg)

### New DB Tables
- [newsengine].[fact_content_embedding] — vector storage (384-dim JSON arrays)
- [media].[fact_video_transcode_job] — transcode job queue
- Index on [newsengine].[fact_feed_user_content_view] — speeds up 3 view history queries
