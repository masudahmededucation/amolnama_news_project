1. ADVANCED QUESTION TYPES (3 types)
-------------------------------------
Status: NOT STARTED
Estimated effort: 2-3 days each
Priority: LOW — only if BCS / certified-exam users ask

a) calculated  — math expression with variable substitution
   - DB: new coll_question_calculated_var table (var_name, min, max, step)
   - Engine: substitute random var values per session, evaluate formula,
     match student's numeric answer within tolerance
   - Use case: math / physics / chemistry exam prep

b) drag-drop on image  — student places labels on an image hot-spot
   - DB: new coll_question_image_hotspot table (x_percent, y_percent,
     radius_percent, label_text)
   - Engine: collect (label_id, x, y) tuples; check each landed within radius
   - Use case: anatomy, geography, parts-of-a-machine

c) regex / pattern  — fill-blank but answer matched against a regex
   - DB: extend coll_question_option with option_match_pattern column
   - Engine: re-use _grade_fill_blank with re.fullmatch fallback
   - Use case: code / language quizzes


2. QUIZ TEMPLATE MARKETPLACE
----------------------------
Status: NOT STARTED
Estimated effort: 3-5 days
Priority: MEDIUM — network effect if users share quiz templates

Vision:
  Staff or quiz-creators can publish a quiz as a "template". Others
  can browse the catalogue and clone it as a starting point.

DB:
  - coll_quiz_template_listing
      (link_quiz_id, listed_by_user_profile_id, listing_status_code,
       template_category_code, template_description, clone_count, listed_at)
  - All cloning already works via the existing /api/exam/<id>/clone/ endpoint

UI:
  /quizadmin/quiz-templates/  — gallery, filter by topic / language,
                                  click "Clone to my drafts"
  Quiz form — new "Publish as template" toggle in the Status section

Auth:
  - Anyone with edit access on a quiz can publish it as template.
  - Cloning requires quiz creator permission.


3. COMMENT MENTIONS + EMAIL DIGEST
-----------------------------------
Status: NOT STARTED
Estimated effort: 1-2 days
Priority: LOW — wait for users to actually use comments first

Use case:
  In a quiz comment, type "@username" — auto-link to their profile +
  drop a pulse notification ("X mentioned you in a quiz comment").

Tech:
  - Reuse existing newsengine.notifications.create_notification.
  - Parse @mentions on save (regex against sanitised HTML).
  - Use existing UserProfile.display_name index for resolution.

5. STUDENT FLAG / DISPUTE A QUESTION
-------------------------------------
Status: DONE (2026-04-17)

Shipped:
  - 🚩 "Report a problem" button on the quiz-take explanation panel.
    Reasons: wrong_answer, unclear_question, wrong_source, duplicate, offensive, other.
    Optional 500-char note. Auto-archives the question after 3 pending reports.
  - Staff inbox at /quizadmin/flagged/ with status filter
    (pending / resolved / invalid / all). Inline Resolve / Reject actions.
  - APIs: POST /mastermind/api/question/report/  (already existed)
          GET  /mastermind/api/question/report/list/
          POST /mastermind/api/question/report/<id>/review/  body {action: resolve|reject}

7. EXPORT GRADEBOOK TO CSV
---------------------------
Status: DONE (2026-04-17)

Shipped:
  - GET /mastermind/api/exam/<quiz_id>/gradebook.csv  (?include_in_progress=1)

8. PERIODIC BACKUP-EVERYTHING JSON DUMP
----------------------------------------
Status: DONE (2026-04-17)

Shipped:
  - GET /mastermind/api/admin/full-backup/  (staff only)
  - Streams: topics, books, chapters, questions (with options + match-pairs +
    resolved topic_code/difficulty_code), quizzes (with pool entries)
  - Excluded by design (run-state, not authored content): sessions,
    certificates, comments, webhooks, proctoring violations
  - Output is round-trip-safe with the existing import pipeline
  - Cron usage: curl -b cookies https://site/mastermind/api/admin/full-backup/ -o backup.json
