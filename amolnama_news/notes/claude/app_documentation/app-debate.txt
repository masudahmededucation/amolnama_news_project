## Concept
Scheduled, text-based debate platform — "Digital Colosseum".
Structured argument arena, NOT a comment section.
Blue (Pro) vs Red (Against) with crossing-over rebuttals.

## Core Layout
- Header (sticky): Topic title + description + status badge + countdown
- Stats line: "🔥 X জন যুদ্ধে নেমেছেন · Xটি যুক্তি চলছে — আপনিও যোগ দিন!"
- Passion Board: 4 tug-of-war bars (যোদ্ধা, যুক্তি, সমর্থন, বাক্য)
- Join Panel: Custom side labels (e.g. "পার্থ সাহেবের পক্ষে" / "পার্থ সাহেবের বিপক্ষে")
- Three-column arena: Blue board | VS | Red board
- Reply Drawer: centered bottom panel (max-width 600px) with crossing-over color
- Mobile: stacked boards (no forced 3 columns)

## The Crossing-Over Routing Logic (Core Feature)
| Action       | User Team | Target Side   | Logic                              |
|-------------|-----------|---------------|------------------------------------|
| New Argument | Pro       | Pro Board     | Starts a new Pro argument          |
| New Argument | Against   | Against Board | Starts a new Against argument      |
| Reply        | Pro       | Against Board | Replying to opponent = rebuttal    |
| Reply        | Against   | Pro Board     | Replying to opponent = rebuttal    |

Key Rule: A reply NEVER stays on the same side. Reply always = opposition.
Server-enforced: `participant.link_team_side_id == parent_post.link_author_team_side_id` → rejected.
Admin bypass for testing (TODO: remove before production).

## Custom Side Labels
- `blue_side_label` / `red_side_label` on coll_topic (NVARCHAR 200, nullable)
- If empty, falls back to "পক্ষে (Pro)" / "বিপক্ষে (Against)"
- Used in: join panel, board headers, stats bar, passion board
- Set during topic creation with example placeholders

## Garbage Detection (Server-side)
- Minimum character count (default 20, configurable per topic)
- Minimum sentence count (default 1)
- Emoji-only blocked (single emojis in text OK, emoji-only posts blocked)
- Emoji spam blocked (>50% emoji ratio)
- Repeated character spam blocked (>50% ratio)
- Non-language content blocked (>50% ratio)
- Duplicate detection via SHA-256 content hash
- All metrics stored on post: post_character_count, post_sentence_count, post_emoji_ratio, etc.

## Voting System
- Upvote (+1) / Downvote (-1) toggle
- One vote per user per target (unique constraint)
- Same vote again = remove (toggle off)
- Different vote = flip
- Cached counts on post: upvote_count, downvote_count, score
- Cached counts on topic: topic_upvote_count, topic_downvote_count, topic_score

## Passion Board (Live Stats Dashboard)
Per-side cached aggregates on coll_topic:
- blue/red_participant_count — যোদ্ধা (Fighters)
- blue/red_post_count — যুক্তি (Arguments)
- blue/red_upvote_count — সমর্থন (Support)
- blue/red_sentence_count — বাক্য (Sentences)
- blue/red_character_count — (stored but not displayed yet)

Visual: dark gradient background, tug-of-war bars with blue/red glow.
Bars pulse when one side dominates (>70%).

## Search
- Client-side instant filter on debate home page
- NFC normalization for Bengali text comparison
- Strips chandrabindu (ঁ), anusvara (ং), visarga (ঃ) for fuzzy matching
- Multi-word search: each word must match somewhere in title or description
- Partial match: typing সংবি finds সংবিধান

## Rotating Placeholders
- 12 Bengali respect/quality reminders rotate on focus
- Applied to: blue composer, red composer, reply drawer
- Examples: "সবাইকে সম্মান দিয়ে কথা বলুন", "ব্যক্তি আক্রমণ নয়, যুক্তি দিন"
