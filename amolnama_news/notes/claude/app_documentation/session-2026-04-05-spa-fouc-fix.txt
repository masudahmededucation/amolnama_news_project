Session 2026-04-05 — SPA FOUC (Double Flash) Prevention
========================================================

Problem: Moving from MPA (Multi-Page App) to SPA caused "double flash" —
user sees unstyled content for a split second before CSS loads and content shifts.

Root Cause: CSS loading timing during SPA <main> swap, NOT hydration mismatch.
This is a Django SSR + vanilla JS app. React/Vue/Inertia.js do NOT apply.

WHAT WAS ALREADY DONE (commit ba6b8ba)
---------------------------------------
- spa-navigation.js: loadPageCss() returns Promise
- Content hidden (opacity:0) during swap
- Reveal only after cssReadyPromise.then() + 500ms timeout fallback
- requestAnimationFrame wraps opacity reveal

GAPS FOUND AND FIXED (this session)
-------------------------------------

GAP 1: CSS PREFETCH ON HOVER
  File: spa-navigation.js (lines 170-186)
  Problem: HTML prefetched on hover, but page-specific CSS only loaded after parse
  Fix: preloadPageCss() function — parses prefetched HTML, injects
       <link rel="preload" as="style"> for page-specific CSS
  Result: CSS already in browser cache when user clicks — zero CSS wait time

GAP 2: FIRST-LOAD CLOAK (MPA FOUC protection)
  File: base.html (lines 44-54)
  Problem: SPA swap was protected, but initial Django page load had no cloak
  Fix: Critical inline CSS sets main { opacity: 0 }
       html.spa-ready main { opacity: 1 } — class added on DOMContentLoaded
       800ms timeout fallback — never invisible forever
  Result: First page load fades in smoothly after CSS is ready

GAP 3: OPACITY TRANSITION DURATION
  File: base.css (line 52)
  Problem: 100ms transition too fast to mask micro-flashes
  Fix: Changed from .1s to .15s (150ms)
  Result: Professional-level fade that hides any remaining timing gaps

GAP 4: SCROLL POSITION RESTORE ON BACK/FORWARD
  File: spa-navigation.js (lines 83-87, 112-118, 270-276)
  Problem: Always scrolled to top on back/forward — disorienting
  Fix: Step 0 saves window.scrollY via history.replaceState before navigating
       Step 8 restores scrollY on popstate, scrolls to top on new navigation
       popstate handler passes event.state.scrollY to navigateTo()
  Result: Back/forward returns user to exact scroll position

GAP 5: PRELOAD CLEANUP
  File: spa-navigation.js (lines 210-212)
  Problem: <link rel="preload"> tags accumulate in <head> after multiple hovers
  Fix: Remove all data-spa-preload links before loading new page CSS
  Result: Clean <head>, no stale preload links
