## Introduction
Professional text extraction tool — OCR for images, PDF text extraction,
audio/video transcription. Bengali + English. Supports folder watcher
(auto-process files dropped into input folder) and web upload.

## Architecture

### PDF Processing Flow (Smart Decision Engine)
1. PyMuPDF opens PDF, extracts text directly (instant for digital PDFs)
2. Per-page validation: checks if text is real Bengali or garbled font encoding
3. **Garbled detection**: corruption markers (঳, ঱, ৞া etc.) + common Bengali word check
4. If garbled or scanned (<5 words): Tesseract OCR at 300 DPI with ben+eng
5. Light preprocessing: crop margins → grayscale → sharpen (no aggressive threshold)
6. If OCR fails but direct text exists: use direct text as fallback
7. Output .txt file written to media/textextractor/output/

### Why Tesseract (not EasyOCR/PaddleOCR for PDFs)
- **EasyOCR**: 60+ seconds per page on CPU — too slow for multi-page documents (PyTorch CRAFT+ResNet)
- **PaddleOCR**: Does NOT support Bengali language. Also has PIR/oneDNN crash on Windows CPU.
- **Tesseract**: ~4.5 seconds per page. Official ben.traineddata model. Stable on all Python versions.
- EasyOCR still used for standalone image files (JPG, PNG etc.) where it provides bounding boxes.

### Garbled Bengali Detection
PDFs with subsetted/non-standard fonts produce garbled text (mojibake). Example:
- Garbled: `ফাাংরাদেদ঱য ঳াংবফধান`
- Correct: `বাংলাদেশের সংবিধান`

Detection logic:
1. Count Bengali characters (U+0980-U+09FF)
2. If >25% Bengali chars, check for corruption markers: `঳`, `঱`, `৞া`, `ফাাং` etc.
3. Also check if ANY common Bengali words appear (এবং, করা, হবে, বাংলাদেশ etc.)
4. If corruption markers >8 or zero common words found → garbled → use Tesseract OCR

### Data Storage Strategy
- **DB stores ONLY metadata**: status, word count, page count, confidence, processing time, file paths
- **Extracted text goes to .txt file**: media/textextractor/output/EXTRACTED_{filename}.txt
- **Detail page reads from .txt file**: _read_output_file() in views.py
- **NEVER store large text in DB**: pyodbc sends >4000 char strings as ntext which breaks UTF-8 collation

### Key Modules
- engines/__init__.py — routes file to correct engine by extension
- engines/engine_pymupdf.py — PDF smart extractor: direct text → validate Bengali → Tesseract OCR fallback
- engines/engine_easyocr.py — Bengali+English OCR for images, lazy-loads model once. extract_fast() for detail=0 mode.
- engines/engine_tesseract.py — Tesseract OCR wrapper (ben+eng), light preprocessing
- engines/engine_whisper.py — audio/video transcription, lazy-loads model once
- engines/engine_plaintext.py — plain file reader with encoding fallback (UTF-8 → latin-1)
- engines/engine_paddleocr.py — PaddleOCR sidecar (NOT USED — Bengali not supported)
- ocr_worker.py — PaddleOCR subprocess worker (NOT USED — Bengali not supported)
- processor.py — shared job processing logic (create job, process, save metadata, write output file)
- views.py — dashboard + detail pages, reads extracted text from .txt file
- views_api.py — upload, cancel (sets is_active=False), delete, status APIs

### Smart Preprocessing (built into each engine)
- **Images (EasyOCR)**: auto-resize to max 2000x2000px before OCR
- **PDF pages (Tesseract)**: crop-to-content → grayscale → light sharpen. 300 DPI rendering. No aggressive threshold.
- **Video**: auto-extract audio track via ffmpeg before Whisper
- **Text**: auto-detect encoding (UTF-8, UTF-8 BOM, latin-1 fallback)
- All temp files cleaned up after processing

### Folder Watcher
- Run: python manage.py run_textextractor
- Config from DB: config_folder_watcher table
- Uses watchdog library to monitor input folder
- wait_for_file_copy() prevents reading incomplete files
- Processes in background thread per file

## Features

### Dashboard (/text-extractor/)
- Job list: file name, extension, size, engine, language, confidence
- Processing details: word count, pages, processing time
- Status badges: queued (yellow), processing (blue), completed (green), failed (red)
- Error messages shown inline
- Cancel button: hides job (sets is_active=False)
- Delete button: hides job (sets is_active=False)
- Upload via drag & drop (file-dropzone component) — multi-file supported
- Upload hides all previous jobs automatically

### Job Detail (/text-extractor/job/<id>/)
- Full extracted text read from .txt output file (pre-wrap formatting)
- Copy to clipboard button
- Download as .txt button
- Confidence %, word count, processing time, detected language

## Performance (64-page Bengali PDF benchmark)
- EasyOCR on CPU: ~60 seconds/page (64 pages = ~64 minutes) — UNACCEPTABLE
- PaddleOCR: Bengali NOT SUPPORTED, plus PIR/oneDNN crash on Windows
- Tesseract ben+eng: ~4.5 seconds/page (64 pages = ~291 seconds / ~5 minutes) — PRODUCTION READY

## Troubleshooting
- **Garbled Bengali from PDF**: Font encoding issue. Auto-detected by corruption markers + common word check. Falls back to Tesseract OCR.
- **Tesseract not found**: Install via `choco install tesseract` (Windows) or `apt install tesseract-ocr` (Linux). Verify: `tesseract --list-langs` should show `ben`.
- **EasyOCR first load**: 2-3 seconds to load model (lazy-loaded, cached after). Only used for image files, NOT PDFs.
- **Whisper first load**: downloads model on first use (~150MB for base)
- **NEVER store large text in DB**: pyodbc sends >4000 char strings as ntext which breaks UTF-8 collation
- **NEVER modify db_backend encoding**: corrupts ALL Bengali data to ????
- **PaddleOCR does NOT support Bengali**: Tested PP-OCRv3/v4/v5 — `lang='bn'` throws ValueError. Also has ConvertPirAttribute2RuntimeAttribute crash on Windows CPU. Do not use.
