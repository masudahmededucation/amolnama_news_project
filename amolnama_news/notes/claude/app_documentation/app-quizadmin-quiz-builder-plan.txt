GOLD-STANDARD RESEARCH — HOW DO PROFESSIONAL QUIZ BUILDERS WORK?
-----------------------------------------------------------------

1. GOOGLE FORMS (quiz mode)
   Flow: create → set title + description → add questions one at a time.
   Each question has a type dropdown. Types supported:
     - Short answer, Paragraph (essay)
     - Multiple choice (radio), Checkboxes (multi), Dropdown
     - File upload, Linear scale, Multiple-choice grid, Checkbox grid
     - Date, Time
   For each MCQ/checkbox question, staff add answer options inline and
   flag which are correct (with feedback per option).
   Images: any question type can have an image attached (stem image).
   Setting per form: shuffle question order, shuffle option order,
   show correct answers after submit, send scores by email.

2. KAHOOT
   Flow: title + cover image → add slides (questions). Types:
     - Quiz (4-option MCQ)
     - True/False
     - Puzzle (order answers)
     - Type answer (fill in the blank)
     - Poll / word cloud / open-ended (no correct answer)
   Each question has its own image or video attachment, time limit, points
   weight. Global settings: randomise question + answer order.

3. TYPEFORM / MICROSOFT FORMS / QUIZIZZ
   Same pattern — sequential question builder, per-question type picker,
   per-question image attachment, quiz-level settings for time/shuffle.

COMMON DENOMINATOR (what I'm baking into our builder):
  - Quiz has: title (BN + EN), description, topic, time limit, pass %,
    negative marking, shuffle-questions, shuffle-options, attempts allowed,
    cover image.
  - Questions belong to a quiz; each has a TYPE selected from a dropdown.
  - Each question type renders its own inline editor.
  - Image attachment is a per-question feature, not a question type per se
    (a radio-button question can still have an image in the stem).
  - Drag-reorder of questions within a quiz.
  - "Save draft" vs "Publish" two-step.

DB WRITE LOGIC — ONE TRANSACTION PER SAVE
-----------------------------------------
On save, wrap in transaction.atomic():

  1. Upsert coll_exam row (create if new, update if editing).
  2. For each question in the form:
       a. Upsert coll_question row (link_mastermind_ref_question_type_id,
          link_mastermind_ref_topic_id, link_mastermind_ref_book_id, etc).
       b. Replace coll_question_option rows for MCQ types (delete all +
          re-insert, same approach as my current update_question_manual).
       c. For essay type, no options are created.
       d. Upsert map_exam_question_pool row linking question → exam.
  3. Delete any stale map_exam_question_pool rows for questions removed
     from the form (soft-delete or hard-delete — I'll use is_active=0).

Rationale for "replace wholesale" on options: simpler than diffing,
matches UX (one form submit, one batch).
