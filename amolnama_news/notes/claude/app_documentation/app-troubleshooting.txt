### R0. Quiz Panel review queue scrolled to top on approve/reject
Date: 2026-04-16
Was: Clicking approve/reject on /quizadmin/review/ reloaded the page and
     browser scrolled to top, losing reviewer's visual context.
Root cause: navigateToQuestionId() in page-review-queue.js set
     window.location.search which causes a full reload, and reload
     always scrolls to top. Prev/Next anchor links had the same issue.
Fix: Append #quizadmin-review-card fragment to every navigation URL, so
     the browser automatically scrolls to the review card on load.
     Added id="quizadmin-review-card" to the <article> element. Full path
     is now window.location.pathname + '?' + params + '#quizadmin-review-card'.
Lesson: When a page reload is necessary in a staff workflow where the
     user's visual focus is mid-page, use URL fragments (#anchor) so the
     browser restores focus. Don't let the reviewer chase the UI.

### R16. JSON island in HTML can leak </script> from user-controlled fields
Date: 2026-04-17
Was: Mastermind multi-player lobby pages rendered server-side state via
<script type="application/json">{{ payload_json|safe }}</script>. The payload
contains player.display_name from UserProfile. A malicious display_name
containing </script> would break out of the JSON block.
Fix: Switched to Django's built-in {{ payload|json_script:"id" }} filter
which auto-escapes </ → \u003c. View must pre-stringify any datetime /
Decimal values because json_script can't serialise them — added
mastermind/views._stringify_dates() helper that walks the dict/list and
ISO-formats datetimes, floats Decimals.
Location: mastermind/views.py, mastermind/templates/mastermind/pages/lobby_player.html,
quizadmin/templates/quizadmin/pages/quiz_host.html, quizadmin/views.py.

### R17. Django ORM __contains substring match has prefix-collision bug
Date: 2026-04-17
Was: lobby.submit_lobby_answer used __contains=f'"question_index": {N}' to
detect duplicate submissions. The substring "question_index": 1 also matches
"question_index": 10 / 11 / 12 / etc., so a player who answered question 1
would be falsely blocked from answering questions 10+.
Fix: Anchor the match with a trailing comma — f'"question_index": {N},'.
Works because answer_submitted event payloads always have question_index
followed by other fields (is_correct, points_earned), so the comma is
guaranteed to be present in the JSON output.
Location: mastermind/lobby.py, submit_lobby_answer().

### R18. .toUpperCase() on identifier input is OK; on display data is not
Date: 2026-04-17
Was: Bug-scan flagged .upper() / .toUpperCase() in lobby.py + views.py for
join codes. CLAUDE.md rule says no case conversion anywhere.
Decision: KEPT for join codes — they are identifiers from a deliberately
uppercase-only alphabet (BCDFGHJKLMNPQRSTVWXYZ23456789), not user-facing
display data. Normalising user input to match the stored case is required.
The rule applies to status_code / display data, not fixed-alphabet identifiers.
Location: mastermind/lobby.py:_generate_unique_join_code, get_lobby_state.

### R21. Web Audio API — non-essential sound that must never break the app
Date: 2026-04-17
Was: Lobby pages need sound effects (correct, wrong, time-warning, etc.)
without external audio files (CSP-safe + free). Sound is non-essential —
gameplay must continue if the audio context fails.
Pattern (mastermind/static/mastermind/assets/js/lobby-sfx.js):
  - AudioContext created lazily on first play() call (browser autoplay policy
    requires user gesture — don't construct on script load)
  - Each sound is composed at play time as a tiny synth pattern
    (oscillator + gainNode with smooth attack/release envelopes — no clicks)
  - Three intentional silent catches: localStorage blocked (private mode),
    AudioContext.resume() failed (autoplay policy), _playToneSequence threw
    (browser quirk). All annotated. Sound is explicitly non-essential so
    swallow + comment is the correct design (NOT a bug).
  - Per-event 150ms debounce so spammed state messages don't blast the user.
  - Honours prefers-reduced-motion AND localStorage("mastermind_lobby_sfx")
    so users can mute persistently.
  - Module exposes wireToggleButton(buttonElement) — single source of truth
    for wiring 🔊 / 🔇 toggle buttons (used by both lobby-player.js and
    lobby-host.js with one line each, no duplication).
Location: mastermind/static/mastermind/assets/js/lobby-sfx.js.

### R24. Long Bengali text + raw pyodbc INSERT pattern (chunk_text fix)
Date: 2026-04-17
Was: Saving chapter text via the new book editor needed to insert long
Bengali strings into coll_book_chunk.chunk_text (NVARCHAR(MAX), UTF-8
collation). ODBC Driver 17 promotes long parameters to SQL_WLONGVARCHAR
(ntext), which fails the _SC collation cast.
Fix: Reuse the proven pattern from ai_generator.ingest_book_from_pdf —
drop to the raw pyodbc cursor on the Django connection, set
cursor.setinputsizes() with (pyodbc.SQL_WVARCHAR, 0, 0) on the chunk_text
parameter only, INSERT, commit. See site_apps/mastermind/book_editor.py
function _raw_insert_book_chunk(). Same fix as the original PDF chunker
— do NOT extract it to a "shared utility" because the column index in
setinputsizes is positional and fragile. Better to copy + comment.
Location: site_apps/mastermind/book_editor.py.

### R25. Django |linebreaks vs custom paragraph splitting in templates
Date: 2026-04-17
Was: Initial book_reader.html template used a non-existent

### R26. Public chapter-text endpoint — separate from edit endpoint
Date: 2026-04-17
Was: The reader page needed to lazy-load chapters as the visitor clicks

### R27. Imported book has chunks but no chapters → "No chapters yet"
Date: 2026-04-17
Was: ai_generator.ingest_book_from_pdf creates rows in coll_book_chunk
but does NOT auto-create rows in coll_book_chapter. The book editor's
chapter sidebar reads from coll_book_chapter — so for a book imported
via PDF (where the user never manually added chapters), the editor
showed "No chapters yet" even though 66 pages of text existed in chunks.
Fix: book_editor.ensure_default_chapter_for_orphan_chunks(book_id) —
idempotent migration that creates ONE default chapter and bulk-updates
every orphan chunk to point to it. Called from book_editor_page on every
visit (no-op when chapters exist). Result: legacy imported books open in
the editor with all their text immediately accessible.
Location: site_apps/mastermind/book_editor.py
ensure_default_chapter_for_orphan_chunks() + quizadmin/views.py
book_editor_page() entry block.

### R28. Hardcoded is_staff_user=False breaks staff edits on legacy data
Date: 2026-04-17
Was: book_editor.add_chapter_to_book had `is_staff_user=False` baked
into the call to _user_can_edit_book. For a legacy book where
link_created_by_user_profile_id is NULL (pre-dates the column), the
permission check returned False even when the calling user was staff.
Net effect: 400 Bad Request on every "+ Add chapter" click.
Fix: Add is_staff_user parameter to add_chapter_to_book; the API
endpoint extracts it from request.user.is_staff and passes through.
Same pattern applied to update_chapter_metadata and the new
update_book_metadata / delete_chapter / reorder_chapter functions.
Lesson: never bake permission flags into helper signatures — always
accept them from the caller so the request context stays the source of
truth. A staff bypass is fine, but it has to ACTUALLY honour staff.
Location: site_apps/mastermind/book_editor.py + views_api.py.

### R29. CSS custom-property theming via [data-reader-theme="..."]
Date: 2026-04-17
Was: Book reader v2 needed light / sepia / dark themes that switch
instantly without page reload AND cascade onto every element inside the
reader (background, text, borders, accent, muted text). Class swapping
(adding/removing .theme-dark) requires every styled element to know about
the theme class — fragile.
Pattern: Set CSS custom properties on the root reader element gated by
[data-reader-theme="sepia"] and [data-reader-theme="dark"] attribute
selectors. The base rule defines all variables (--reader-bg, --reader-text,
--reader-muted, --reader-border, --reader-pane-bg, --reader-accent).
Theme overrides re-set ONLY those variables. Every child uses
var(--reader-*), so swapping the data attribute on the root re-cascades
the entire palette.
Bonus: same pattern works for font-size — [data-reader-font-size="..."]
gates a single rule that sets font-size on .reader-pane.
Location: mastermind/static/mastermind/assets/css/components/book-reader.css.

### R30. Engine functions accept is_staff_user explicitly (no implicit context)
Date: 2026-04-17
Was: The pure-engine modules (mastermind/book_editor.py) used to default
is_staff_user=False so a forgetful caller would get the safer behaviour.
Bit me badly when add_chapter_to_book was called from an API view that
didn't pass it through — see R28.
Pattern: For every engine helper that gates on is_staff_user, the
parameter MUST be supplied explicitly by the caller. The default value
remains False for safety in shell tests, but every API endpoint is
responsible for passing `bool(request.user.is_staff or
request.user.is_superuser)`. Document in the function docstring.
Location: site_apps/mastermind/book_editor.py.
