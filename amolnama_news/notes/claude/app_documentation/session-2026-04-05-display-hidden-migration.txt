Session 2026-04-05 — display-hidden to hidden Attribute Migration
==================================================================

Migrated CSS class .display-hidden to HTML hidden attribute across
entire project. 230 files changed across all 15 apps.

PROBLEM
-------
.display-hidden { display: none } is a CSS class in forms.css.
Before CSS loads, the class does nothing — overlays, modals, hidden
panels flash on page load. HTML hidden attribute works immediately
because browser default [hidden] { display: none } is built-in.

PROVEN PATTERN (photo-lightbox, post-feed-new-posts-pill)
---------------------------------------------------------
HTML: <div class="photo-lightbox" id="photo-lightbox" hidden>
CSS:  .photo-lightbox:not([hidden]) { display: flex; }
JS:   lightboxElement.hidden = false; // show
      lightboxElement.hidden = true;  // hide

MIGRATION SCOPE
---------------
355 HTML template instances (class="display-hidden" -> hidden attribute)
373 JS instances (classList.add/remove/toggle/contains -> .hidden property)
12 CSS selectors with :not([hidden]) added (where explicit display rules exist)
