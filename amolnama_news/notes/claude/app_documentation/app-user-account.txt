## Introduction
User authentication and profile management. Custom User model (email-based,
no username). Person + UserProfile + contact models. Device/session tracking.

## Features

### Authentication
- Email-based login (no username field on User model)
- Signup: creates User + Person + UserProfile atomically (transaction.atomic)
- Device/session tracking on login
- django-axes for brute-force protection

### User Profile
- display_name: shown everywhere in the UI
- username_handle: auto-generated @username from display name (core/utils.py)
- bio: short description for hover popup
- about: longer description for profile page
- contribution_score_count: +10 per upvote received on posts
- is_verified: verified badge on all posts
- avatar: uploaded via portal app

### Context Processors
- user_display_name: injects display_name from UserProfile into all templates
- user_avatar_url: injects avatar URL into all templates

## Key Functions

### Views
- login_view() — email + password authentication
- signup_view() — atomic User + Person + UserProfile creation
- logout_view() — session cleanup

### APIs
- api_language_pref() — save BN/EN preference (called by language toggle)

## Templates (13)
- login.html, signup.html, password reset forms, etc.

## Troubleshooting
- Custom User model: email-based, first_name/last_name removed from User — names in Person only
- Signup: MUST be in transaction.atomic() — prevents orphan records
- Person creation: check all NOT NULL columns (first_name_en, modified_at)
- Phone storage: contact.phone stores country_code + phone_number separately; account.user.phone stores full number
- normalize_phone: regex bug was fixed — don't re-add regex
- Non-mandatory text columns: use `or None` not empty string
