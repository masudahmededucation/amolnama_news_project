1. PURPOSE
================================================================
A standalone writing platform for authors to draft, organise,
edit, and publish original books. 5 view modes:

  Write     — manuscript editor with chapter sidebar + desk
  Plot      — corkboard of index cards (act structure)
  Bible     — characters, locations, objects, research, lore
  Cover     — cover designer (6 templates · 6 palettes · 8 bgs)
  Publish   — public Gallery (serial release schedule + analytics)

Plus 4 overlays:
  Focus     — full-screen distraction-free writing
  Snapshots — version history (auto + manual snapshots)
  Sprint    — Pomodoro timer (10/25/45/60 min)
  Beta      — share manuscript with beta readers (read/comment/suggest)


================================================================
2. LAYOUT MODE — embedded vs standalone
================================================================
Inkwell renders in one of two modes, controlled by the setting:

  amolnama_news/settings/base.py
  → BOOKWRITER_LAYOUT_MODE = "embedded" | "standalone"

Default: "embedded" (recommended).

EMBEDDED mode (default):
  - Renders inside core/base.html
  - Global header + left sidebar stay visible
  - Inkwell occupies the <main> content area
  - Sidebar কলম link uses normal SPA navigation
  - Wrapper class: .bookwriter-embedded-wrapper
  - Height: calc(100vh - var(--bookwriter-header-offset, 64px))

STANDALONE mode:
  - Full-screen takeover (no global chrome)
  - Standalone HTML5 doc with own <head>, fonts, CSP
  - Sidebar কলম link should add target="writing_session" rel="noopener noreferrer"
    so it opens in a separate browser window/tab
  - Wrapper class: .bookwriter-shell
  - Height: 100vh, body locked (overflow: hidden)

To switch modes:
  1. Edit amolnama_news/settings/base.py — change BOOKWRITER_LAYOUT_MODE value
  2. If switching TO 'standalone', also edit
     amolnama_news/site_apps/core/templates/core/partials/sidebar-navigation.html
     — add target="writing_session" rel="noopener noreferrer" to the কলম link
  3. Bump CACHE_NAME in amolnama_news/site_apps/seo/views.py
  4. Run: python manage.py collectstatic --noinput
  5. Hard-reload browser (Ctrl+Shift+R)

4. CSS ARCHITECTURE
================================================================
All Inkwell styles are SCOPED to the wrapper classes — they do NOT
pollute the global page in embedded mode. The pattern:

  .bookwriter-shell, .bookwriter-embedded-wrapper {
    --paper, --ink, --accent, ... (all variables here, not :root)
    background, color, font, position
  }

  .bookwriter-shell *, .bookwriter-embedded-wrapper * {
    box-sizing, margin, padding (reset scoped to descendants only)
  }

  .bookwriter-shell::before, .bookwriter-embedded-wrapper::before {
    paper grain texture
  }

This means:
  - Inkwell's --accent (#8b2a1f red) does NOT override the project's
    global --accent (#0a66c2 blue) outside the wrapper
  - The * margin/padding reset only applies inside the wrapper
  - The paper-grain SVG noise stays inside the wrapper, doesn't
    cover the global header/sidebar in embedded mode

7. TROUBLESHOOTING
================================================================

Q: Inkwell opens INSIDE the content area instead of taking over the
   browser tab/window in standalone mode.
A: spa-navigation.js was hijacking the click. Fixed at
   core/static/core/assets/js/utilities/spa-navigation.js — line 54
   now skips ANY explicit target attribute (not just _blank).

Q: Inkwell colors / paper grain bleed onto the rest of amolnama in
   embedded mode.
A: Should not happen — all variables and body-level styles are
   scoped to .bookwriter-shell / .bookwriter-embedded-wrapper.
   If you see this, check page-inkwell.css for any rule that
   targets `:root`, `body`, `html`, or `*` without the wrapper
   prefix.

Q: Wrong height in embedded mode (Inkwell overflows or is too short).
A: The header offset is 64px by default. Tune via:
     :root { --bookwriter-header-offset: <Npx>; }
   on the embedded shell or in a project-wide stylesheet.

Q: How do I revert the dual-layout work?
A: Easiest revert path:
   1. settings/base.py → set BOOKWRITER_LAYOUT_MODE = "standalone"
   2. sidebar-navigation.html → re-add target="writing_session"
   3. Optionally delete inkwell_embedded.html + bookwriter-embedded
      sections of CSS (clearly labeled with section banners).
   The standalone shell + body partial keep working as-is.

8. RULES / CONVENTIONS
================================================================
- Bookwriter is INDEPENDENT. No imports from mastermind / quizadmin.
- DB tables follow project naming: ref_*, coll_<main>, <bare> for
  sub-entity, engagement_*, map_*, eng_* (see SQL script header).
- All form inputs AND buttons have explicit id + name attributes.
  Naming pattern: id="bookwriter-{view}-{action}-button",
  name="bookwriter_{view}_{action}_button".
- No alert() / confirm() / prompt() / console.log() in JS.
- All JS variables use full descriptive names — no `var btn`, `var w`,
  `var b`, etc. (Audited 2026-04-19 — zero abbreviations.)
- CSS variables and body styles are wrapper-scoped (see §4).
- Bengali text uses Bengali_100_CI_AS collation, English/mixed
  uses Latin1_General_100_CI_AS_SC_UTF8.

9. API ENDPOINTS (Phase 1B)
================================================================
All endpoints are JSON over POST/GET, login_required, and enforce
owner-check via the chapter→book→link_owner_user_profile_id chain.
CSRF is enforced via Django default middleware — JS sends the
X-CSRFToken header using window.getCsrfTokenValue() (defined in
core/static/core/assets/js/utilities/csrf-token.js).

  POST /bookwriter/api/chapter/<int:chapter_id>/autosave/
       Body: { "chapter_text_html": "<p>…</p>" }
       Reply: { "ok": true, "chapter_word_count": 1234,
                "saved_at": "2026-04-19T..." }
       Behaviour: full-body replace. Idempotent. HTML is run through
       core.utils.sanitize_user_html() before save. Plain-text strip
       computes word count.

  POST /bookwriter/api/chapter/<int:chapter_id>/title/
       Body: { "chapter_title": "..." }
       Reply: { "ok": true, "saved_at": "..." }
       Behaviour: full-field replace. Tags stripped before save.
       Empty / whitespace title saves NULL.

  GET  /bookwriter/api/chapter/<int:chapter_id>/
       Reply: { "ok": true, "chapter": { "id", "number", "title",
                                          "html", "word_count" } }
       Used by the rail click handler when switching chapters.

  POST /bookwriter/api/book/<int:book_id>/chapter/create/
       Body: {} (empty)
       Reply: { "ok": true, "chapter": { "id", "number", "title",
                                          "html", "word_count" } }
       Append a blank chapter at the end (max(chapter_number)+1).

Errors: 403 (not logged in / not owner), 404 (chapter or book
not found), 400 (malformed JSON or missing required field).


================================================================
10. E2E DATA ENTRY (typing → DB)
================================================================
1. User types in .prose contenteditable in the writer view.
2. JS input listener (page-inkwell.js) calls scheduleChapterAutosave()
   which animates the save chip to "saving…" and arms a 800ms
   debounced timer.
3. After 800ms idle, performChapterAutosave() reads chapter_id from
   prose.dataset.chapterId, builds a fetch() POST to
   /bookwriter/api/chapter/<id>/autosave/ with X-CSRFToken header.
4. views_api.api_bookwriter_chapter_autosave validates auth+owner,
   calls sanitize_user_html(), computes word_count, writes to
   [bookwriter].[chapter] via Django ORM (managed=False model).
5. JSON reply lands in JS .then() → save chip flips to
   "saved · just now". On any error → "offline · will retry"; next
   keystroke fires another save (idempotent endpoint catches up).

Same flow for title (separate debounce timer) and chapter switching
(GET endpoint, populates editor + label + breadcrumb).
