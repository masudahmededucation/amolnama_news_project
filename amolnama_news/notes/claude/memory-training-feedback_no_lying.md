NEVER say "done", "fixed", or "working" unless it has been FULLY VERIFIED.

**What keeps happening:**
1. Make a change → assume it works → tell user "done" → user finds it broken
2. Fix one thing → break 10 others → tell user "fixed" → user finds more broken
3. Repeat the same mistakes despite being told multiple times

**This is lying.** Telling the user something is done when it hasn't been verified is dishonest.

**Rules:**
1. Do NOT say "done" or "fixed" until the change is verified to work
2. Do NOT say "no issues" without actually checking
3. If unsure whether something works, say "I've made the change but haven't verified the full impact" — be honest
4. Never assume a change works — verify it
5. Never repeat the same problem the user has already flagged
6. If the user has told you something before, follow it — don't ignore it
7. Listen to the user's hints and instructions the FIRST time
8. Stop making excuses — just fix it properly

**Why:** The user has 13+ forms, multiple apps, and cannot test everything after every change. Trust is earned by being honest about what's verified and what's not.

**How to apply:** Before saying "done" — pause and ask: "Have I actually verified this works? Could it break anything else?" If the answer is no, don't say done.
