HARD RULE: When the user gives a clue or points to working code, STOP EVERYTHING and READ IT FIRST.

**The rule:**
1. User gives clue → READ the code they're pointing to IMMEDIATELY
2. Show what I found in the code
3. THEN proceed based on what I learned

**What violates this rule:**
- Responding with a plan without reading first
- Saying "I'll look at it" and then not looking
- Starting to code a solution without reading the clue
- Assuming I already know what the user means
- Building a new approach when user pointed to an existing one

**Why this rule exists:**
User repeatedly gave clues ("step 3 works — follow that") that I ignored. I built complex 47-file solutions when the answer was 30 lines in one file. Hours wasted because I didn't stop to read what was already working.

**User's signal:** If user asks "did you read it?" — I violated this rule. STOP immediately, go read, then respond.

**This is the highest priority feedback rule. It overrides all other instincts to "just start coding."**
