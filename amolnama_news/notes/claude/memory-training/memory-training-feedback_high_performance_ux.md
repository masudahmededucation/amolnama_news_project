name: High-performance UX standard
description: When building new features/cards, follow professional app benchmarking and ultra-fast architecture principles
type: feedback
---

When building new features, cards, or pages:

1. Check existing professional apps and benchmark features found in industry-leading applications
2. Prioritize "light" and ultra-fast architecture — offload heavy processing to web workers and async threads
3. Prevent screen freezing, mouse stuttering, or browser lag by isolating process execution from the UI
4. Ensure zero-learning-curve UX — anticipate user needs with clean, frictionless layout
5. Refine all inputs for professional grammar and clarity with silent self-correction
6. Execute all complex calculations and data tasks as invisible background work
7. Keep UI fluid and responsive at all times, mimicking elite professional software
8. Deliver polished results through progressive rendering and optimized resource management
9. All technical "work" must remain invisible — the experience should feel effortless and instantaneous

**Why:** User wants every new feature to match the quality bar of professional/commercial applications. No amateur or slow-feeling implementations.
**How to apply:** Every time a new card, tool, page, or feature is created — research similar professional implementations first, then build with performance and UX as top priorities. Use Web Workers for computation, async for data, progressive rendering for display.
