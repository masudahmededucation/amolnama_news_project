--i have seen one website has lyrics. 
not able to right click. 
not able to see the text using inspect. how are they doing it?

They're likely using a combination of techniques:

Canvas rendering — The lyrics text is drawn on an HTML <canvas> element instead of regular DOM text. You can see it but it's not selectable text — it's pixels, like an image. Inspect element shows just <canvas>, no text content.

SVG text with user-select disabled — Text rendered as SVG paths. Looks like text but isn't copyable.

CSS user-select: none — Prevents text selection (but inspect element still shows the text).

JavaScript event blocking — Disable right-click, Ctrl+C, Ctrl+U, F12. (But developers can bypass this.)

Web font obfuscation — Custom font where characters are mapped to wrong Unicode points. The DOM shows gibberish but the custom font renders it correctly. Inspect shows meaningless characters.

Method 1 (Canvas) is the strongest — the text literally doesn't exist in the DOM. Even "View Source" shows nothing. The lyrics are fetched from the server and painted directly onto canvas.
